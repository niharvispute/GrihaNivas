Part A — Builder Submission Flow
How a Builder Requests a Project Listing
Before the admin creates anything, the builder discovers GrihaNivas and submits a project application through the public website.
1
Builder clicks "LIST PROPERTY" in the header
On any page of the GrihaNivas website, the builder clicks the LIST PROPERTY button visible in the top navigation. This opens the property listing wizard.
The List Property wizard — Step 1 of 5, showing listing type options: SALE, RENT, BULK PROJECT
2
Builder selects "BULK PROJECT"
The wizard shows three listing types — SALE, RENT, and BULK PROJECT. For a builder with a multi-unit development (tower, society, plotted development), they select BULK PROJECT. This immediately switches to a dedicated project application form — skipping the standard individual property steps.
Why BULK PROJECT? Individual SALE/RENT listings go through a 5-step property form suited for single units. BULK PROJECT is specifically designed for builders/developers listing an entire project with multiple units and configurations.

Part A — Step 03
Submission Confirmation
After the builder submits the form, they immediately see a confirmation screen.
"Application Received!" — the builder sees this confirmation immediately after submitting

What the confirmation says: "Thanks for telling us about your project. Our team will personally contact you within 24 hours to take this forward."

The submission is now live in the Admin CRM. No further action is needed from the builder — the GrihaNivas team takes over from here.
Response commitment: The platform confirms a 24-hour callback window. Ensure the admin team reviews new PROJECT APPLICATION leads daily so this SLA is met.
Part A — Step 04

Part A — Step 06
Admin Contacts the Builder
Contact is made externally — by phone, WhatsApp, or email — using the contact details from the lead. The goal is to collect all remaining details needed to create the full project listing.
1
Advance the lead to "CONTACTED"
Before calling, click → CONTACTED on the lead row to update the status. This signals to the team that this lead is being worked on.
2
Collect full project details from the builder
During the call, gather all the information needed for the 5-step admin listing wizard. Use this checklist:
  • Project name, builder name, and contact person
  • RERA registration number and MahaRERA URL
  • Complete address and pincode
  • Number of towers, floors, and total land area
  • All BHK configurations — carpet area range, bathrooms, balconies, units per config
  • Price range (starting price, max price, price per sq.ft)
  • Possession timeline and project status (New Launch / Under Construction / Ready)
  • High-res images — hero image, gallery photos, floor plans per BHK, master plan
  • Project brochure (PDF)
  • YouTube/Vimeo video link (if available)
  • Amenities and highlights
3
Log the call outcome in Activity Notes
After the call, open the Lead Detail panel and add a note — e.g. "Called Rahul Mehta. Confirmed 2BHK (720–850 sqft) and 3BHK (1050–1250 sqft). Images to be shared by email by 28 Jun. Price ₹1.25–2.2 Cr."
4
Advance lead to "QUALIFIED" once details are received
Once all images, pricing, and documents are in hand, click → QUALIFIED on the lead. This means the project is ready to be listed. Proceed to Part B.
Important: Do not start creating the project listing until you have at least: project name, location, BHK configurations with unit counts, starting price, and a hero image. A listing without these will appear incomplete to users.
Moving to Part B
Admin Creates the Project Listing
Now that the admin has collected all details from the builder, they log into the Admin Console and create the full listing using the 5-step project wizard.
Part B — Step 07
Logging into the Admin Console
Access the platform at grihanivas.in/login and sign in with your admin credentials.
1
Go to the Login page
Open your browser and navigate to the platform URL. Click Login if not already selected. You will see fields for email/mobile and password.
The GrihaNivas login page — Account Access
2
Enter admin credentials
Type your registered mobile number (without the +91 country code prefix) or email, and your password. Then click Login.
Important: Enter the mobile number as 10 digits only (e.g. 9876543210), not with +91 prefix.
3
Access the Admin Console
After login, click My Account in the top navigation or navigate directly to /admin. You will land on the Admin Dashboard.
Admin Dashboard — showing key stats: leads, properties, users, blogs
Tip: The Admin Console sidebar gives you quick access to all sections — Properties, Projects, Leads, Blogs, Banners, Users, and more. The Dashboard is your overview at a glance.
Part B — Step 08
Navigating to Projects
The Projects section manages all bulk housing projects (not individual resale/rental listings).
1
Click "Projects" in the sidebar
In the left sidebar, click the domain Projects menu item. This takes you to the Projects management page which lists all existing projects.
Projects list — shows all projects with status, configs, unit count, and quick-edit actions
2
Click "+ Add Project"
At the top right of the projects page, click the + Add Project button. This opens the 5-step project creation wizard.
Projects vs Properties: Projects are large-scale developments (e.g., Lodha Skyline — 200 units across 4 towers). Properties are individual units for buy/rent listings. Use Projects for builder launches, Under Construction, and New Launch developments.

Step 4 — Pricing & Inventory
Set the overall project price range and optionally manage individual unit inventory.
Step 4 of 5 — Price summary, per-configuration pricing, and unit inventory management

A
Project Price Summary
Enter the starting price, maximum price, price per sq.ft, and maintenance charges. These populate the price range shown on the public listing card.

B
Configuration Pricing (Optional)
Click Edit Pricing next to each BHK type to set a specific price range per configuration. If left blank, the listing shows "Price on Request" for that config.
C
Unit Inventory (Optional)
Add individual units manually via + Add Unit, or bulk import using the CSV/XLSX template. This enables per-unit availability tracking (Available / Hold / Booked / Sold).
Tip: You can skip unit inventory for now and add it later by editing the project. The listing will still go live with the configuration-level data.
Part B — Step 13
Step 5 — Review & Publish
Final review of all details, SEO settings, lead capture toggles, and publishing the listing live.
Step 5 of 5 — Full review summary with SEO preview and lead capture settings

1
Review the Project Summary
Check the auto-generated summary: Project Name, Builder, RERA Number, Location, Type, Status, and Configuration table with unit counts.
2
Configure Lead Capture Settings
Toggle on/off which enquiry options appear on the public page:
  • Enable Price Request
  • Enable Callback
  • Enable Brochure Download
  • WhatsApp CTA Enabled
  • Enable Site Visit
3
Check SEO Settings
The SEO Title, Meta Description, and URL Slug are auto-generated from the project name and location. You can edit them if needed. The Google Search preview shows exactly how it will appear in search results.
4
Set Status to "Published" and Click Publish
Change the Listing Status dropdown from Draft to Published, then click Publish Project. The listing goes live immediately.
Status set to "Published" — click "Publish Project" to go live
5
Confirmation — Project is Live
After publishing, you are redirected to the Projects list. The new project appears at the top with status Active, listing type, configurations, and unit count visible.
Lodha Skyline now appears in the Projects list as Active — 2BHK, 3BHK · 200 units

Once published, the project immediately appears on the public-facing Projects page for all users to browse.
1
User visits the Projects page
Any visitor to /projects will see the new listing. The project card shows the builder name, location, BHK types, starting price, and a VIEW PROJECT button.
Public Projects page — Lodha Skyline appears at the top as a New Launch with ₹1.25 Cr starting price
What users see on the card: NEW LAUNCH badge · Project name · BHK types · Builder · Location · Starting price · VIEW PROJECT button

Project Detail Page
The full project detail page users see when they click VIEW PROJECT.
Lodha Skyline detail page — RERA verified badge, price range, configurations, enquiry form, and builder info

Hero Section

Part C — Step 16
User Submits an Enquiry
The user fills in the enquiry form on the project detail page — this creates a lead in the admin CRM.
1
Fill the Enquiry Form
If logged in, Name, Email, and Phone are auto-filled from the user's profile. The user types their message and clicks Send Inquiry.
Enquiry form pre-filled with user details and a custom message about 2 BHK floor plans
Phone format: The form requires a 10-digit number without the +91 country code (e.g. 9172008630).
2
Confirmation Message
On successful submission, a green success message appears: "Inquiry submitted! Our team will contact you shortly." The form resets.
Success confirmation — "Inquiry submitted! Our team will contact you shortly."

Advance the Lead Status
Click the → CONTACTED button on any NEW lead to immediately advance it. The button label updates to reflect the next stage (→ QUALIFIED, → CLOSED). There is also an undo button to revert the status.
Lead advanced to CONTACTED — next action shows "→ QUALIFIED", with undo option
